# 掃碼維修功能 - 使用者操作手冊

## 功能簡介

掃碼維修功能是對既有「料槍維修」流程的優化升級。通過掃碼槍或手動輸入料槍編號，系統自動帶出料槍基本資訊，加快維修開單速度，同時減少人為輸入錯誤。

---

## 快速開始

### 步驟 1：進入掃碼維修頁面

1. 開啟料槍維修系統
2. 點擊主頁上的「**掃碼維修**」按鈕（紅色「料槍送修」按鈕旁）

### 步驟 2：掃碼或手動輸入料槍編號

輸入框會自動獲得焦點（無需手動點擊）。

#### 方式 A：使用掃碼槍（推薦）
1. 準備掃碼槍
2. 掃入料槍上的條碼
3. 系統自動送出 Enter 鍵，觸發查詢

#### 方式 B：手動輸入
1. 在輸入框中輸入料槍編號
2. 按下 **Enter 鍵**
3. 系統查詢料槍資訊

### 步驟 3：系統自動帶出料槍資訊

查詢成功後，以下欄位會自動填入（**唯讀，無法修改**）：
- **設備編號** (Eno)
- **料槍編號** (Sno)
- **廠商** (Trade)
- **型號** (Size)

### 步驟 4：填寫維修資訊

在「維修資訊」區域填寫以下欄位：

| 欄位 | 必填 | 說明 |
|------|------|------|
| **檢修不良分類** | ✅ | 從下拉選單選擇，支援搜尋 |
| **檢修不良原因** | ✅ | 從下拉選單選擇，支援搜尋；若選「其他」需填寫原因說明 |
| **更換部品名稱** | ❌ | 若有更換零件，請填寫名稱 |
| **更換部品料號** | ❌ | 若有更換零件，請填寫料號 |
| **其他說明** | ❌ | 當選擇「其他」原因時，此欄才會顯示，需填寫詳細說明 |

#### 使用 Select2 下拉選單

- **搜尋**：點擊下拉框後輸入關鍵字，系統會自動篩選選項
- **清除**：已選擇後，點擊 X 按鈕可清除選擇

### 步驟 5：儲存維修記錄

1. 確認所有必填欄位已填寫
2. 點擊 **[儲存]** 按鈕
3. 系統會驗證並儲存
4. 顯示成功提示，1.5 秒後自動返回主頁

## 常見情況與排查

### 情況 1：查詢失敗 - 料槍不存在

**錯誤訊息**：「查無料槍編號：XXX」

**原因**：
- 輸入的料槍編號在系統中不存在
- 條碼掃入錯誤
- 料槍已報廢

**排查方式**：
1. 檢查料槍編號是否正確
2. 確認料槍編號未包含特殊符號或多餘空白
3. 檢查料槍是否已報廢（聯絡主管確認）
4. 嘗試在「料槍送修」中手動查詢該料槍

### 情況 2：查詢失敗 - 料槍無待維修紀錄

**錯誤訊息**：「料槍 XXX 無待維修紀錄或已全部完修」

**原因**：
- 該料槍已全部完修，沒有待維修紀錄
- 料槍從未送修過

**排查方式**：
1. 若確實需要維修，請先進入「料槍送修」頁面開單
2. 等待幾秒鐘重試，可能存在資料同步延遲
3. 聯絡系統管理員確認料槍狀態

### 情況 3：儲存失敗

**錯誤訊息**：「儲存失敗」或伺服器相關錯誤

**排查方式**：
1. 檢查網路連線是否正常
2. 確認所有必填欄位已填寫
   - 檢修不良分類：必選
   - 檢修不良原因：必選
   - 若原因為「其他」，需填寫說明
3. 嘗試刷新頁面重新操作
4. 如問題持續，聯絡 IT 部門報告錯誤訊息

### 情況 4：頁面焦點異常

**症狀**：輸入框未獲得焦點，無法直接掃碼

**排查方式**：
1. 手動點擊輸入框
2. 檢查瀏覽器是否為最新版本
3. 清除瀏覽器快取（Ctrl + Shift + Delete）並重試

### 情況 5：下拉選單無法搜尋

**症狀**：Select2 下拉框沒有顯示搜尋框

**排查方式**：
1. 刷新頁面
2. 檢查瀏覽器 JavaScript 是否啟用
3. 如問題持續，聯絡 IT 部門

```
進入掃碼維修頁面
  ↓
輸入料槍編號（掃碼或手動）
  ↓
按 Enter 鍵
  ↓
系統查詢料槍資訊 ─→ 查詢失敗 ─→ SweetAlert2 錯誤提示
  ↓ (查詢成功)          ↓
  ├→ 清空輸入框
自動帶出：           ├→ 焦點回輸入框
  - 設備編號            └→ 重新輸入
  - 料槍編號
  - 廠商
  - 型號
  ↓
填寫維修資訊：
  - 檢修不良分類（必填）
  - 檢修不良原因（必填）
  - 更換部品名稱（選填）
  - 更換部品料號（選填）
  ↓

  ↓
系統驗證 ─→ 驗證失敗 ─→ SweetAlert2 提示缺少必填欄位
  ↓ (驗證成功)  ↓

SweetAlert2 成功提示
  ↓

  ↓
完成！
```

## 最佳實踐

### ✅ 推薦做法

1. **使用掃碼槍**
   - 掃碼槍自動送出 Enter，速度快
   - 減少人為輸入錯誤

2. **立即填寫維修資訊**
   - 料槍帶出後應立即填寫，提高效率
   - 不要中斷填寫流程

3. **使用 Select2 搜尋功能**
   - 選項眾多時，輸入關鍵字篩選
   - 比滾動下拉框更快找到目標

4. **定期檢查網路連線**
   - 掃碼失敗可能與網路延遲有關
   - 若 API 查詢超時，稍後重試

### ❌ 避免做法

1. **不要手動修改自動帶出的欄位**
   - 這些欄位 disabled，無法修改
   - 確保資料準確性

2. **不要連續快速掃碼**
   - 給系統充分時間查詢和顯示
   - 建議間隔 1 秒以上

3. **不要在多標籤頁同時使用掃碼維修**
   - 可能造成焦點衝突
   - 建議只在一個標籤頁操作

## 鍵盤快捷鍵

| 快捷鍵 | 功能 |
|--------|------|
| **Enter** | 提交掃碼/輸入或儲存表單 |
| **Tab** | 移動焦點至下一個欄位 |
| **Shift + Tab** | 移動焦點至上一個欄位 |
| **Escape** | 關閉 SweetAlert2 提示（取決於提示類型） |

## 故障排查決策樹

```
是否能進入掃碼維修頁面？
├─ 否 → 檢查登入狀態，確認帳戶權限
├─ 是 → 
    是否能掃碼/輸入料槍編號？
    ├─ 否 → 檢查瀏覽器 JavaScript 是否啟用
    ├─ 是 →
        查詢是否成功？
        ├─ 否 (料槍不存在) → 檢查料槍編號，確認料槍未報廢
        ├─ 否 (無待維修) → 進入「料槍送修」先開單
        ├─ 是 →
            是否能填寫維修資訊？
            ├─ 否 → 刷新頁面，檢查瀏覽器相容性
            ├─ 是 →
                是否能儲存？
                ├─ 否 → 檢查必填欄位，檢查網路連線
                ├─ 是 → 操作成功！
```
