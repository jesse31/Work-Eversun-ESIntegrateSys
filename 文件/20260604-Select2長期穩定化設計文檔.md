**建立日期**: 2026-06-04  
**修改工件**: `ESIntegrateSys/Views/MaterialGun/ScanRepair.cshtml`  
**影響範圍**: 掃碼維修頁面 Select2 初始化策略

## 技術決策記錄

### D1: 非同步等待策略（Promise-based 輪詢）

**決策**: 實作 `ensureSelect2Ready(timeout = 1000)` 方法，使用 async/await + while loop 輪詢 `window.$.fn.select2`

**實作**:
```javascript
async ensureSelect2Ready(timeout = 1000) {
    const startTime = Date.now();
    const checkInterval = 50;
    
    while (Date.now() - startTime < timeout) {
        if (window.$ && window.$.fn.select2) {
            console.log('[ScanRepair] Select2 已就緒');
            return true;
        }
        await new Promise(resolve => setTimeout(resolve, checkInterval));
    }
    
    return false;
}
```

**優勢**:
- 簡潔清晰的邏輯，易於理解和維護
- 每 50ms 檢查一次，平衡精度與性能
- 1000ms 超時確保在開發版 Vue 下也能準備好

**風險緩解**:
- 若 Select2 CDN 完全失敗，超時 1000ms 後自動返回 false，進入降級路徑

---

### D2: 非同步啟動（不阻塞 mounted）

**決策**: 在 `mounted()` 中使用 `setTimeout(async () => {...}, 0)` 啟動 Select2 初始化，不等待完成

**實作**:
```javascript
mounted() {
    this.$refs.scanInputRef.focus();
    
    // 非同步啟動（推至事件循環末尾，讓 Vue 優先渲染頁面）
    setTimeout(async () => {
        const ready = await this.ensureSelect2Ready();
        if (ready) {
            this.loadDropdownOptions();
        } else {
            this.select2Available = false;
            console.error('[ScanRepair] Select2 超時（1000ms 內未加載），降級為原生 select');
        }
    }, 0);
}
```

**優勢**:
- 用戶可立即掃碼（聚焦輸入框），Select2 後臺準備
- 不阻塞 Vue 的頁面渲染

**風險緩解**:
- 若用戶在 Select2 準備前掃碼，下拉選單會降級為原生 select（功能完整但無搜尋）

---

### D3: 降級為原生 HTML select

**決策**: 當 Select2 超時時，自動跳過 Select2 初始化，保留原生 HTML select

**實作**:
```javascript
success: (data) => {
    this.classificationOptions = data || [];
    
    // 守衛檢查
    if (!window.$ || !window.$.fn.select2) {
        console.warn('[ScanRepair] Select2 未就緒，分類下拉降級為原生 select');
        return; // 不執行 Select2 初始化
    }
    
    // Select2 就緒，正常初始化
    this.$nextTick(() => {
        $('#Classification').select2({ allowClear: true, ... });
    });
}
```

**優勢**:
- 用戶體驗降級但不中斷：仍可選擇維修分類和原因
- 原生 select 完整支援所有項目（雖然無搜尋功能）
- 項目數量有限（<50），用戶可快速掃描

**風險緩解**:
- 無搜尋功能可能增加用戶的操作步驟，但在項目數量有限的情況下仍可接受

---

### D4: AJAX 回調中的守衛檢查

**決策**: 每個 AJAX success callback 中重新檢查 `window.$.fn.select2` 是否可用

**理由**:
- `mounted()` 中檢查 Select2，但 AJAX 回應可能延遲 200-300ms
- 在這期間 Select2 可能加載失敗或頁面被卸載
- 防禦性編程確保安全

**實作**: 分類和維修原因 AJAX 的 success callback 中都添加守衛檢查

## 降級路徑

```
頁面載入
    ↓
mounted() → setTimeout(..., 0) 啟動 ensureSelect2Ready()
    ↓
ensureSelect2Ready() 輪詢 1000ms
    ↓
Select2 就緒?
├─ YES → loadDropdownOptions() → AJAX 取選項
│   ├─ Classification AJAX success → 守衛檢查 → Select2 初始化 ✓
│   └─ MResult AJAX success → 守衛檢查 → Select2 初始化 ✓
│       ↓
│   用戶見 Select2 搜尋欄 ✓
│
└─ NO (超時 1000ms) → select2Available = false
    ↓
    loadDropdownOptions() 不執行（或執行但跳過 Select2 初始化）
    ↓
    原生 HTML select 被 Vue 綁定
    ↓
    用戶見原生下拉，可正常選擇（無搜尋）✓
```

## 驗證與測試

### 測試案例 1：正常流程（Select2 就緒）
- 頁面載入 → ensureSelect2Ready() 找到 Select2（通常 <50ms）→ loadDropdownOptions()
- Classification 和 MResult AJAX 成功 → Select2 初始化
- 用戶見 Select2 搜尋欄，可搜尋和選擇 ✓

### 測試案例 2：降級流程（Select2 超時）
- 使用瀏覽器開發工具 Network throttle（禁用 Select2 CDN）
- ensureSelect2Ready() 輪詢 1000ms 後返回 false
- select2Available = false，console.error 記錄
- loadDropdownOptions() 執行，但守衛檢查攔截 Select2 初始化
- 原生 select 被 Vue 綁定，用戶可正常選擇 ✓

### 測試案例 3：已完修流程
- 掃碼已完修料槍 → API 返回 status: 'all_completed'
- queryGun() 彈警告「此料槍已完修」，表單隱藏
- 無論 Select2 狀態如何，流程正確處理 ✓

### 測試案例 4：表單提交與必填驗證
- 選擇維修分類和原因（Select2 或原生 select）
- formData 同步正確
- 若選擇「99（其他）」，顯示文字輸入框
- 表單驗證成功後提交 ✓
